# セキュリティ規約(公開リポジトリ)

本リポジトリは公開される。git 履歴は不可逆であり、一度コミットされた情報は削除しても拡散済みとみなす。よって**事前予防**を最優先とする。

## 規約

1. **機密値の混入禁止**: API キー / トークン / 秘密鍵 / パスワード / OAuth クレデンシャルを、コード・ドキュメント・コミットメッセージ・テスト fixture・ログの貼り付けに含めない。本プロジェクトは設計上、機密値を一切必要としない(ネットワーク通信ゼロ・外部サービス連携なし)。**機密値らしき文字列が現れた時点で、それ自体を異常として作業を停止しユーザーに報告する**
2. **実パスの禁止**: `/Users` 始まりの絶対パス(ユーザー名を含むパス)を書かない。`~` または `$HOME` で表記する。対象はソースコード・docs・example・コメント・コミットメッセージの全て
3. **合成データのみ**: テスト fixture、スクリーンショット、README / docs の例示に使う HTML・ファイル名・データは全て合成(架空)とする。実在の個人生成 HTML やその内容をリポジトリに持ち込まない
4. **実設定ファイルのコピー禁止**: `~/.claude/settings.json` などの実設定をコピー・転記して example を作らない。example は必要最小限を手書きする(実設定には他ツールの設定や認証情報が含まれうるため)
5. **検知時の手順**: push 前に発見 → 該当コミットを履歴書き換えで除去してから push。push 後に発見 → 該当クレデンシャルを即 revoke → 履歴書き換え + force-push → ユーザーへ報告(revoke が先、隠蔽が後)
6. **コミット author**: 個人メールアドレスではなく GitHub noreply アドレスを使う(リポジトリローカルの `git config user.email` 設定済み)

## 機械的検証

- `make check` が規約 2(実パス)と .gitignore の機能を検証する。コミット前に必ず実行する
- public 化時に GitHub の Secret scanning + Push protection を有効化する(規約 1 の構造的バックストップ)

## sunset 条件

- 規約 2 はリポジトリを private 化したら緩和してよい
- 規約 1 / 3 は恒久(private でも維持)
