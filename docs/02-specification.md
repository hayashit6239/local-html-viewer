# 02. 仕様書(外部仕様)

最終更新: 2026-06-12(初版。UI 確定の M7 / M9 で追補する)

## 1. 画面構成

シングルウィンドウ(2 ペイン)。デザインリファレンス: `assets/design-mock-b.html`(案 B: アンバー × macOS ネイティブ・ハイブリッド)。

```
┌────────────┬──────────────────────────────┐
│ sidebar     │ topbar: ファイル名 + 所属パス │
│  検索       │         再読込 / Finder /     │
│  フォルダ一覧│         ブラウザで開く        │
│  [最近|ツリー]├──────────────────────────────┤
│  ファイルリスト│ プレビュー (WKWebView)        │
│  監視ステータス│   角丸カードで白背景表示       │
└────────────┴──────────────────────────────┘
```

- **sidebar**: 検索フィールド / 登録フォルダ一覧(live インジケータ付き、+ で追加)/ 「最近」「ツリー」セグメント / ファイルリスト / フッタに監視状態と件数
- **最近タブ**: 全登録フォルダ横断で mtime 降順のフラットリスト。直近更新ファイルにバッジ表示
- **ツリータブ**: フォルダ階層表示
- **検索**: インクリメンタル。クエリがあれば両タブともフィルタ結果を表示
- **プレビュー**: 選択ファイルを WKWebView で表示。背景は白(コンテンツ忠実性優先、ダークテーマを注入しない)

## 2. 挙動

| イベント | 挙動 |
|---|---|
| 起動 | 登録フォルダを走査し、最新 mtime のファイルを自動表示 |
| ファイル選択 | 即プレビュー(クリック / ↑↓ / j k) |
| 表示中ファイルが変更された | 自動リロード(スクロール位置はベストエフォートで維持)+ 控えめな通知表示 |
| 表示中ファイルが一覧から消えた | 最新ファイルを再選択(残り 0 件なら空状態)。ライブ削除検知は M6、M4 では次回 rescan 時に反映 |
| 新規 .html がフォルダに出現 | リストに自動反映(最近タブ先頭) |
| 登録フォルダ外のファイルを外部から受信 | プレビュー表示し、リスト先頭に EXTERNAL バッジ付きでピン留め。フォルダの自動登録はしない |
| 外部 http(s) リンクをクリック | 既定ブラウザで開く(アプリ内では遷移しない) |
| `mailto:` / `tel:` / `facetime:` / `sms:` リンクをクリック | OS の既定ハンドラへ委譲(メーラー・電話等が起動) |
| フォルダが見つからない(外付け等) | missing バッジ表示。勝手に登録解除しない |

## 3. キーボード

| キー | 動作 |
|---|---|
| `↑` `↓` / `j` `k` | 選択移動(即プレビュー) |
| `/` | 検索フィールドへフォーカス |
| `Esc` | 検索クリア・リストへフォーカス復帰 |
| `r` | 表示中ファイルを再読込 |
| `⌘⇧R` | Finder で表示 |

検索フィールドフォーカス中は `j` `k` `r` をテキスト入力として扱う(グローバルキーにしない)。

## 4. 外部連携(`open` / Claude Code hook)

- `open -b com.hayashi.htmlviewer <file.html>` で、未起動なら起動して表示、起動中なら同一インスタンスが表示を切り替える(多重起動しない)
- Dock アイコンへの .html ドラッグ&ドロップでも同様に開く
- **Claude Code hook**(`hooks/open-html.sh` + `hooks/settings.json.example`):
  - PostToolUse(Write / Edit)で `.html` / `.htm` の生成・編集を検知
  - `open -g -b com.hayashi.htmlviewer <file>` で**フォーカスを奪わずに**アプリへ渡す
  - 同一ファイルの連続イベントは 5 秒間スロットル
  - 表示中ファイルへの再 open はアプリ側で no-op(中身の更新反映は live reload が担当)

## 5. 制約・既知の挙動

- **ローカル完結**: アプリ本体は通信ゼロ。ただし表示する HTML 自身が外部リソース(CDN 等)を参照する場合は WKWebView がそれを取得する(コンテンツ起因の通信)
- `file://` オリジンの `fetch` / XHR はブラウザ同様 CORS で失敗する(self-contained HTML を前提とする)
- 表示ファイルの所属フォルダ外への `file://` 参照は読み込まれない(セキュリティスコープ)
- スクロール位置の復元はベストエフォート(JS レンダリングが重いページでは完全には戻らないことがある)
- 初回アクセス時に macOS がフォルダアクセスの許可を求めることがある(TCC)。再ビルド後に走査結果が空になった場合の対処は `03-implementation.md` を参照
